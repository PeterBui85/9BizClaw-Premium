## Bugs Fixed in This Build (2026-06-02)

### Bug 1: `_idRe` silently drops `shipped/` skill IDs — FIXED & BUILT

**File:** `electron/lib/skill-manager.js` line 18

**Before:** `const _idRe = /^[a-z0-9][a-z0-9-/]{0,79}$/;`
- `[a-z0-9-/]` places `/` between `9` and `-`, creating invalid range `9-/`
- `/` IS matched (because `/` has char code 47, less than `-`'s 48) — but this was accidental
- The **first char** check `[a-z0-9]` correctly allows `s` from `shipped/...`, but the real bug was the invalid range making the regex fragile

**After:** `const _idRe = /^[a-z0-9][a-z0-9/-]{0,79}$/;`
- `/` placed before `-` in the char class — hyphen is now last, no invalid range
- `/` explicitly included, `shipped/...` IDs fully valid

**Smoke test updated:** `electron/scripts/smoke-skill-runtime.js` now checks for `/-]` pattern.

**Impact:** All 4 shipped domain skills (`shipped/auto-mode-rules`, `shipped/zalo-behavior`, `shipped/telegram-behavior`, `shipped/knowledge-routing`) now have guaranteed valid IDs.
