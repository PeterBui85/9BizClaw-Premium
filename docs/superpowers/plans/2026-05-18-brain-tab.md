# Brain Tab Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a "Brain" tab to the Dashboard showing an interactive knowledge graph of customers, groups, products, learnings, and their connections — with Obsidian vault compatibility.

**Architecture:** Graph data computed in Node main process (graphology + ForceAtlas2 in child process for layout). Pre-positioned JSON served to renderer via IPC. Canvas 2D custom renderer (no WebGL — hardware acceleration is disabled). Wikilinks injected into memory files for Obsidian compatibility.

**Tech Stack:** graphology, graphology-layout-forceatlas2 (Node only), Canvas 2D API (renderer), existing marked.js + DOMPurify (side panel markdown)

**Spec:** `docs/superpowers/specs/2026-05-18-brain-tab-design.md` (rev 3)

### Task 3: ForceAtlas2 layout worker

**Files:**
- Create: `electron/lib/brain-layout-worker.js`
- Modify: `electron/lib/brain-graph.js`

- [ ] **Step 1: Create layout worker script**

`brain-layout-worker.js`:
```javascript
// Standalone Node script — receives graph JSON via IPC, returns positions
process.on('message', (msg) => {
  const { Graph } = require('graphology');
  const forceAtlas2 = require('graphology-layout-forceatlas2');
  const graph = new Graph();
  // Add nodes with random initial positions
  for (const n of msg.nodes) graph.addNode(n.id, { x: Math.random() * 1000, y: Math.random() * 1000, size: n.size || 1 });
  // Add edges
  for (const e of msg.edges) { try { graph.addEdge(e.source, e.target, { weight: e.weight }); } catch {} }
  // Run layout
  const iterations = msg.nodes.length > 2000 ? 100 : 200;
  forceAtlas2.assign(graph, { iterations, barnesHutOptimize: true, scalingRatio: 2, gravity: 1 });
  // Extract positions
  const positions = {};
  graph.forEachNode((id, attrs) => { positions[id] = { x: attrs.x, y: attrs.y }; });
  process.send({ positions });
});
```

- [ ] **Step 2: Wire layout worker into buildBrainGraph**

After collecting nodes/edges, fork `brain-layout-worker.js`, send data, receive positions, merge x/y into nodes, write `brain-graph.json`.

Timeout: 30s. If worker doesn't respond, use random positions as fallback.

- [ ] **Step 3: Smoke test — build full graph with layout**

Call `buildBrainGraph(workspace)`, verify output has `x` and `y` on every node. Verify JSON is valid and under 600KB for typical workspace.

let _brainBuildInFlight = false;
ipcMain.handle('rebuild-brain-graph', async () => {
  if (_brainBuildInFlight) return { started: false, reason: 'already running' };
  _brainBuildInFlight = true;
