> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship 100 small UX, performance, code quality, and security improvements in a single overnight session.

**Architecture:** Pure frontend changes in dashboard.html + targeted backend fixes in lib/*.js + smoke test additions. No IPC changes, no new preload bridges unless noted. All changes are additive — no deletions of working features.

**Tech Stack:** HTML/CSS/JS (dashboard.html), Node.js (lib/*.js), shell (smoke tests)

**Execution strategy:** 4 lanes: (A) dashboard.html changes — sequential, one subagent; (B) lib/*.js changes — parallel subagent; (C) smoke tests — after A+B complete; (D) final build. Dashboard.html is a monolithic 11K-line file — concurrent edits will conflict, so all dashboard batches run in sequence.
