fix: split prepare-vendor job per architecture for Mac build

The single "prepare-vendor" job hardcoded TARGET_ARCH=arm64, but the build
matrix runs both arm64 and x64. The x64 build was receiving the arm64
vendor bundle (embedding model + native binaries), producing a DMG with
wrong-architecture vendor files.

Fix by splitting into two prepare jobs:
- prepare-vendor-arm64: TARGET_ARCH=arm64, uploads vendor-bundle-arm64
- prepare-vendor-x64:  TARGET_ARCH=x64,  uploads vendor-bundle-x64

Each build job downloads its matching vendor-bundle-<arch> artifact. The
build matrix runs both arm64 and x64 in parallel.
